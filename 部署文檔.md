# Telegram IT BOT — 部署文檔

本專案**以 HidenCloud「NODE: Budget Free」等免費方案為常見目標環境**設計（Long Polling、單埠後台 + API、SQLite、小記憶體可調 `NODE_OPTIONS`）。下列流程亦適用於其他 **Node.js 20+** 主機。**授權**：見根目錄 **`LICENSE`**（**禁止商業用途**，PolyForm Noncommercial 1.0.0）。

**首選步驟**：請直接依 **`HidenCloud免費主機部署教學.md`** 操作（含官方 **部署時依 `package.json` 自動安裝** npm／yarn、週續約、資源限制；無自動安裝時之 Docker 備援）。本檔補充通用變數、**pm2**、備份與故障排除。

---

## 1. 前置需求

| 項目 | 說明 |
|------|------|
| Node.js | **20 LTS** 或以上（`package.json` 的 `engines`）；**執行主機**必須已安裝 |
| npm | **建置端**需要（用於產生 `node_modules`）；若主機僅執行已上傳的專案，可不裝 npm |
| Telegram | 已由管理員向 @BotFather 取得 **Bot Token** |
| 網路 | 主機需能連線 `api.telegram.org` 及文件中列出的外部 API |

## 2. 取得程式與安裝依賴

### 2.1 主機上可以執行 npm（一般情況）

於專案根目錄（含 `package.json` 之目錄）執行：

```bash
npm install
```

建議在正式環境使用 `npm ci`（需先提交 `package-lock.json` 後再使用）。

### 2.2 主機無法執行 npm install，或無「部署時自動安裝」

**HidenCloud 免費託管**：官方 FAQ 說明，只要專案含 **`package.json`**，在 **deployment** 時會**自動安裝** npm 套件，**npm 與 yarn** 皆可。請優先依 **`HidenCloud免費主機部署教學.md` 第 3 節**，通常**不必**手動上傳 `node_modules`。

若你使用的是**其他**主機，或 HidenCloud 上**僅能手動上傳檔案**且從未觸發會執行 `npm install` 的流程，可改在**自己的電腦或 CI** 先把依賴裝好，再**整包上傳**。

**建議步驟（備援）：**

1. 在可連網的環境取得與線上相同的專案原始碼（git clone 或 zip）。
2. 在該環境執行 `npm install`，產生完整的 **`node_modules`**。
3. 將專案目錄打包（**含** `node_modules`、**含** `package.json`；`.env` 可不在壓縮檔內，上線後再在主機單獨建立）。
4. 上傳並解壓到執行目錄，於主機編輯 `.env`，再以 `node index.js`（HidenCloud 請設 **MAIN_FILE=index.js**）或 `pm2` 啟動（與下文各節相同）。

**務必注意：`better-sqlite3` 為原生模組**

- 在 **Windows** 上執行的 `npm install` 產生的二進位，**不能**直接放到 **Linux** 主機使用，啟動會失敗。
- 若線上為 **Linux（常見）**，請擇一方式在 **Linux 環境** 產生 `node_modules`：
  - **WSL2（Ubuntu）**：在 WSL 內的專案路徑執行 `npm install`，再從 Windows 打包該目錄上傳；或
  - **Docker（建議，有安裝 Docker Desktop 即可）**：在專案根目錄執行（將 `node_modules` 寫回目前目錄）：

    ```bash
    docker run --rm -v "%CD%:/app" -w /app node:20-bookworm-slim npm install
    ```

    PowerShell：

    ```powershell
    docker run --rm -v "${PWD}:/app" -w /app node:20-bookworm-slim npm install
    ```

  - **GitHub Actions**：在 `ubuntu-latest` job 裡 `npm ci`，把整個目錄（含 `node_modules`）打成 artifact 下載後再上傳主機。

若主機為 **Alpine（musl）**，請在 **相同發行版／libc** 的環境安裝依賴，或改用與官方映像標籤一致的 Docker 映像執行上述 `docker run`。

## 3. 環境變數（`.env`）

1. 複製範本並編輯：

   ```bash
   copy .env.example .env
   ```

   （Linux / macOS：`cp .env.example .env`）

2. 必填欄位：

   | 變數 | 說明 |
   |------|------|
   | `ADMIN_USERNAME` | 後台登入帳號 |
   | `ADMIN_PASSWORD` | 後台登入密碼（請使用強密碼） |
   | `JWT_SECRET` | JWT 簽章用密鑰（長隨機字串） |
   | `BOT_TOKEN` | Telegram Bot Token（**首次啟動建議必填**） |

3. 選填／建議：

   | 變數 | 預設 | 說明 |
   |------|------|------|
   | `PORT` | `3000`（程式預設；**請改為主機配發或你指定的埠**） | HTTP 後台與 REST API 埠號 |
   | `DATABASE_PATH` | `./data/app.db` | SQLite 檔案路徑（目錄會自動建立） |
   | `GOOGLE_SAFE_BROWSING_API_KEY` | 無 | `/safe` 指令用；亦可僅在後台設定 |

**注意**：`.env` 勿提交至版本庫（已列於 `.gitignore`）。

## 4. 首次啟動與白名單

1. 啟動程序後，以瀏覽器開啟後台（**根路徑或 /admin 皆可**）：

   `http://<主機>:<PORT>/` 或 `http://<主機>:<PORT>/admin`

   例：`http://<你的網域或 IP>:<PORT>/`（`<PORT>` 須與 `.env` 及主機面板一致）

2. 使用 `.env` 中的管理員帳密登入。

3. 在「白名單」分頁手動新增你的 **Telegram 數字 User ID**（可透過非白名單時 Bot 回覆中的 `{user_id}` 取得，或由其他 Bot 查詢）。

4. 若已在後台寫入 `telegram_bot_token` 且未使用環境變數 `BOT_TOKEN`，**重啟程序**後會讀取資料庫內 Token（見下節）。

## 5. 更新 Bot Token 與 pm2 重啟（穩定性建議）

依 `開發文檔.md` 第 3 節：**不要在執行中熱切換 Token**，更新後請 **重啟整個 Node 程序**。

本專案附 `ecosystem.config.cjs`，預設程序名稱為 **`telegram-it-bot`**。更新 Token 並儲存後請執行：

```bash
pm2 restart telegram-it-bot
```

若你變更了 `ecosystem.config.cjs` 內的 `name`，請改用該名稱。

## 6. 使用 pm2 常駐

全域安裝 pm2（若尚未安裝）：

```bash
npm install -g pm2
```

於專案根目錄啟動：

```bash
pm2 start ecosystem.config.cjs
pm2 save
pm2 startup
```

`pm2 startup` 會印出需以 root 執行的一行指令，請照抄執行以開機自啟。

常用指令：

```bash
pm2 status
pm2 logs telegram-it-bot
pm2 restart telegram-it-bot
pm2 stop telegram-it-bot
```

## 7. 防火牆與埠號

- 對外開放 **`PORT`**（預設程式內為 `3000`，**實際以 `.env` 與主機為準**）供管理後台與 `/api/*` 使用。
- Bot 使用 **Long Polling**，**不需**對 Telegram 開放自訂埠的 Webhook。
- 若僅內網管理後台，請改由 VPN / SSH 隧道存取，並勿將 `ADMIN_PASSWORD` 設為弱密碼。

## 8. SQLite 備份

資料庫預設為 **`DATABASE_PATH` 所指之 `.db` 檔**。WAL 模式下會同目錄產生 `-wal`、`-shm` 檔。

建議備份方式：

- 於**低流量時段**或先 **停止程序** 後，複製整個 `data/` 目錄（含 `.db`、`.db-wal`、`.db-shm`）；或  
- 停止程序後僅複製 `.db`（較單純）。

---

## 9. HTTPS（選用）

本程式僅內建 **HTTP**。若需 HTTPS：

- 於前面加 **Nginx / Caddy** 等反向代理終止 TLS，再轉發至 `127.0.0.1:<PORT>`（與 `.env` 一致）；或  
- 使用主機商提供之應用層 SSL（依平台文件）。

---

## 10. 健康檢查與日誌

- **HTTP**：`GET /` 或 `GET /admin` 以 **sendFile** 回傳 `index.html`（**無** 302，避免與代理重導疊加）。  
- **Bot**：啟動成功時標準輸出會出現 `Telegram Bot 已啟動（Long Polling）`。  
- 錯誤可透過 `pm2 logs telegram-it-bot`（或你的程序名稱）查看。

## 11. 故障排除

| 現象 | 可能原因 | 處理 |
|------|----------|------|
| `database is locked` | SQLite 競爭 | 已啟用 WAL／busy_timeout；仍發生時檢查是否有其他程序鎖定同一 `.db` |
| Bot 無回應 | Token 錯誤、網路、或未重啟 | 查 `pm2 logs`；確認 Token；`pm2 restart` |
| 後台 401 | JWT 過期 | 重新登入 |
| `/safe` 無法使用 | 未設定 API Key | 於後台或 `GOOGLE_SAFE_BROWSING_API_KEY` 設定 GCP Key |
| `better-sqlite3` 安裝失敗 | 缺少編譯環境 | Windows 安裝 **Visual Studio Build Tools**；Linux 安裝 `build-essential`、`python3` |
